Tabungan IKI INA — API Documentation

Dokumentasi lengkap untuk seluruh endpoint REST API aplikasi tabungan digital. Semua endpoint memerlukan autentikasi Bearer Token kecuali yang ditandai khusus. Base URL: https://tabungan-iki-ina.vercel.app

📡 REST API ✅ JSON Response 🔐 JWT Auth
🔐
Autentikasi

Hampir semua endpoint memerlukan token autentikasi dari Supabase.

Bearer Token: Setelah login berhasil, sertakan access_token di setiap request dalam header Authorization.
Authorization: Bearer <access_token>
Content-Type: application/json
Format Response Sukses: Semua response sukses mengikuti format berikut:
{
  "success": true,
  "message": "Pesan sukses",
  "data": { /* data payload */ }
}
Format Response Error:
{
  "success": false,
  "message": "Pesan error",
  "errors": null // atau array detail error validasi
}
🔑
Autentikasi

Endpoint untuk registrasi, login, refresh token, dan logout pengguna.

POST/api/auth/registerPublicRegister▼

Mendaftarkan akun pengguna baru. Email verifikasi akan dikirim ke alamat email yang didaftarkan.

FieldTipeWajibKeterangan
namestring✱ YaNama lengkap pengguna. Minimal 2 karakter.
emailstring✱ YaAlamat email valid yang unik.
passwordstring✱ YaMinimal 8 karakter, harus ada 1 huruf kapital dan 1 angka.
usernamestringOpsionalUsername unik. 3–30 karakter.
{
  "name": "Budi Santoso",
  "email": "budi@email.com",
  "password": "BudiPass123",
  "username": "budisantoso"
}

HTTP 201 — Registrasi berhasil

{
  "success": true,
  "message": "Registrasi berhasil! Silakan cek email Anda untuk verifikasi.",
  "data": {
    "user": {
      "id": "uuid-user",
      "email": "budi@email.com",
      "name": "Budi Santoso",
      "username": "budisantoso",
      "avatar": null
    },
    "session": null
  }
}
201Registrasi berhasil. Email verifikasi terkirim.
400Validasi gagal — nama terlalu pendek, email tidak valid, password tidak memenuhi syarat.
POST/api/auth/loginPublicLogin▼
⚠️Rate limited: maksimal 5 percobaan per 15 menit per IP. Lebih dari itu akan mendapat HTTP 429.

Login dengan email dan password. Mengembalikan token (access_token) dan refresh_token.

FieldTipeWajibKeterangan
emailstring✱ YaAlamat email terdaftar.
passwordstring✱ YaPassword akun.
{
  "email": "budi@email.com",
  "password": "BudiPass123"
}

HTTP 200 — Login sukses

{
  "success": true,
  "message": "Login sukses!",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "refresh_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "user": {
      "id": "uuid-user",
      "email": "budi@email.com",
      "name": "Budi Santoso",
      "username": "budisantoso",
      "avatar": null
    }
  }
}
200Login berhasil. Token dikembalikan.
400Validasi request gagal.
401Email atau password salah.
429Terlalu banyak percobaan login dari IP ini. Coba lagi setelah 15 menit.

Memperbarui access token yang sudah kedaluwarsa menggunakan refresh token.

FieldTipeWajibKeterangan

  "message": "Token berhasil diperbarui",
  "data": {
    "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "refresh_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  }
}
200Token berhasil diperbarui.
400Refresh token tidak valid atau sudah kedaluwarsa.
POST/api/auth/logout🔐 AuthLogout▼

Menghapus session aktif di server. Request body tidak diperlukan.

{
  "success": true,
  "message": "Logout berhasil. Session di server telah dihapus.",
  "data": null
}
200Logout berhasil.
401Token tidak valid atau tidak disertakan.
